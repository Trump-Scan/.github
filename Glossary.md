# 용어집

| 용어                                    | 정의                                                                                                                                       | 예시 값                                                                                                         | 사용 위치                                                                                                 | 비고                                                                         |
| ------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------ | ----------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------- |
| **채널 (Channel)**                      | 데이터가 실제로 게시된 원본 출처를 의미함. 플랫폼, 퍼블리셔, 공식 웹사이트 등에서 생성된 콘텐츠의 ‘출처 단위’로 사용함. 각 채널은 하나의 수집기(Collector)와 1:1로 대응함.                               | TruthSocial, WhiteHouse.gov, CNN 등                                                                           | Collector 인스턴스 이름(`TruthSocialCollector`, `WhiteHouseCollector` 등), 인증·API 관리, 수집 주기 설정, 중복 판정, 신뢰도 계산 | “채널”은 SNS 용어의 Channel과 혼동될 수 있으나, 본 문서에서는 **데이터 수집 단위의 출처(Source)** 로 한정함. |
| **원본 데이터 (Raw Data)**                 | 각 채널(Channel)에서 수집된 최초 상태의 데이터로, HTML·JSON·RSS 등 다양한 포맷을 파싱해 **구조화된 형태(내용, 제목, 등록일시 등)** 로 저장됨. 변환, 정규화, 필터링 등의 후속 처리가 수행되기 이전의 형태를 의미함. | 내용, 제목(옵션), 등록일시 등                                                                                           | Collector Output, Raw Storage                                                          | 데이터 품질 검증 및 재처리를 위해 보존됨. 별도의 스키마 없이 저장 가능.                                 |
| **분석 데이터 (Analysis Data)**            | 원본 데이터(Raw Data)를 LLM이 분석하여 생성한 결과물. 요약, 키워드 등의 필드를 포함하며, 이후 중복 제거 및 피드 생성 단계에서 사용됨.                                                     | semantic_summary, display_summary, keywords                                                                  | Analysis Layer, Deduplication Layer, Feed Generation Layer                                            | LLM Service의 출력 결과로 생성되며, Analysis Store에 저장됨.                             |
| **의미 요약 (Semantic Summary)**          | 원본 데이터의 의미를 압축한 간결한 요약. 임베딩 입력으로 사용되어 중복 제거와 유사도 계산에 활용됨.                                                                                | “Trump to raise tariffs on China.”                                                                           | Deduplication Layer                                                                                   | LLM이 생성한 영어 요약.                                                            |
| **표시 요약 (Display Summary)**           | 사용자 피드에 표시되는 자연어 요약. 문장 수나 형식에 제한이 없으며, 가독성과 전달력을 중시함.                                                                                   | “트럼프는 중국산 제품에 대한 관세 인상을 시사했다. 시장은 부정적으로 반응했다.”                                                               | Feed Generation Layer                                                                                 | 피드 본문에 사용됨.                                                                |
| **키워드 (Keywords)**                     | 발언이 관련된 분류 태그.                                                                                                                              | semiconductor, automotive, steel, energy, tariff                                                             | Feed Generation, Filtering                                                                            | 피드 검색 및 분류 기능에 사용됨.                                                        |
| **임베딩 데이터 (Embedding Data)**          | 분석 데이터(Analysis Data)의 일부 텍스트(예: 의미 요약)를 임베딩 모델을 통해 벡터 형태로 변환한 데이터. 의미 유사도 계산 및 중복 판정에 사용됨.                                              | [0.12, -0.47, 0.33, ...], cosine_similarity=0.85                                                             | Deduplication Layer                                                                                   | 임베딩 생성기는 Embedding Generator, 저장소는 Vector Store를 사용.                       |
| **중복 데이터 (Duplicate Data)**           | 임베딩 유사도(0.85 이상) 기준으로 기존 데이터와 중복으로 판정된 항목. 새 데이터는 처리 중단되고 중복 이력만 저장됨.                                                                    | embedding_id=912, duplicate_of=874, similarity=0.91                                                          | Deduplication Layer                                                                                   | Duplicate History Store에 저장되어 추적 가능.                                       |
| **통과된 분석 데이터 (Passed Analysis Data)** | 중복 검사를 통과해 피드 저장 단계로 전달되는 분석 데이터. 의미적으로 중복되지 않은 새 콘텐츠로 간주됨.                                                                              | analysis_id=513                                                                                              | Deduplication Layer                                                            | Vector Store에 저장된 후 Feed Store Manager로 전달됨.                                     |
| **피드 데이터 (Feed Data)**                | 중복 검사를 통과한 분석 데이터를 기반으로 생성된 최종 사용자용 콘텐츠 단위. 표시 요약(Display Summary), 키워드, 채널 정보, 등록 시각 등의 메타데이터를 포함함.                                       | feed_id=1024, analysis_id=513, channel="TruthSocial", display_summary="…", created_at="2025-11-07T08:31:00Z" | Deduplication Layer, Feed Store                                                                     | 사용자 인터페이스(UI) 또는 알림 서비스에서 노출되는 최종 데이터.                                     |
